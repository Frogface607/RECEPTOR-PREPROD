# 🚀 RAG Upgrade - Продвинутый векторный поиск

## Что было сделано

### 1. Векторные эмбеддинги (`embeddings.py`)
- ✅ Интеграция с OpenAI embeddings API (`text-embedding-3-small`)
- ✅ Кэширование эмбеддингов в памяти
- ✅ Батчинг для оптимизации API вызовов
- ✅ Косинусное сходство для сравнения векторов

### 2. Умный чанкинг (`chunking.py`)
- ✅ Разбиение по смыслу (по заголовкам, параграфам)
- ✅ Перекрывающиеся чанки для контекста
- ✅ Метаданные чанков (категория, источник, позиция)

### 3. Гибридный поиск (`vector_search.py`)
- ✅ Векторный поиск (семантика) - 70% веса
- ✅ Ключевые слова (точные совпадения) - 20% веса
- ✅ BM25 (классическая релевантность) - 10% веса
- ✅ Комбинированный скор для лучших результатов

### 4. Система индексации (`indexer.py`)
- ✅ Автоматическое разбиение документов на чанки
- ✅ Создание эмбеддингов для всех чанков
- ✅ Хранение в MongoDB (`knowledge_base_chunks`)
- ✅ Отслеживание изменений файлов (хеширование)

### 5. Обновленный поиск (`search.py`)
- ✅ Интеграция всех компонентов
- ✅ Fallback на простой текстовый поиск
- ✅ Поддержка категорий и фильтрации

### 6. Интеграция с чатом (`server.py`)
- ✅ Автоматический поиск по базе знаний в контексте
- ✅ Tool calling для `searchKnowledgeBase`
- ✅ Endpoints для индексации и статистики

## Как использовать

### 1. Первичная индексация базы знаний

```bash
# POST запрос для индексации всех документов
curl -X POST http://localhost:8002/api/knowledge-base/index?force_reindex=true
```

Или через Python:
```python
import requests
response = requests.post("http://localhost:8002/api/knowledge-base/index", params={"force_reindex": True})
print(response.json())
```

### 2. Проверка статистики

```bash
# GET запрос для статистики
curl http://localhost:8002/api/knowledge-base/stats
```

### 3. Использование в коде

```python
from receptor_agent.rag.search import search_knowledge_base

# Простой поиск
results = search_knowledge_base(
    "температурные режимы хранения",
    top_k=5,
    categories=["haccp"],
    db_collection=db.knowledge_base_chunks,
    use_vector_search=True
)

# Результаты содержат:
# - content: текст чанка
# - source: имя файла
# - category: категория
# - score: комбинированный скор релевантности
# - vector_score: скор векторного поиска
# - keyword_score: скор ключевых слов
# - bm25_score: скор BM25
```

## Преимущества новой системы

### До (простой текстовый поиск):
- ❌ Только точные совпадения слов
- ❌ Не понимает синонимы
- ❌ Не учитывает контекст
- ❌ Примитивный чанкинг (500 символов)

### После (векторный RAG):
- ✅ Семантическое понимание запросов
- ✅ Понимает синонимы и контекст
- ✅ Умное разбиение на чанки
- ✅ Гибридный поиск (векторный + ключевые слова + BM25)
- ✅ Более релевантные результаты

## Примеры улучшений

### Пример 1: Синонимы
**Запрос:** "как хранить продукты"
**Старый поиск:** Находит только если есть точное слово "хранить"
**Новый поиск:** Находит "хранение", "сохранение", "температурные режимы" и т.д.

### Пример 2: Контекст
**Запрос:** "нормы для ресторана"
**Старый поиск:** Ищет слова "нормы" и "ресторан" отдельно
**Новый поиск:** Понимает связь между словами и находит релевантные нормативы

### Пример 3: Разные формулировки
**Запрос:** "сколько стоит мясо"
**Старый поиск:** Может не найти, если в базе написано "цена на мясо"
**Новый поиск:** Находит по смыслу, даже при разных формулировках

## Технические детали

### Модель эмбеддингов
- **Модель:** `text-embedding-3-small`
- **Размерность:** 1536
- **Стоимость:** ~$0.02 за 1M токенов
- **Скорость:** Быстрая (оптимизирована для production)

### Параметры чанкинга
- **Размер чанка:** 1000 символов
- **Перекрытие:** 200 символов
- **Минимальный размер:** 100 символов

### Веса гибридного поиска
- **Векторный:** 70% (семантика)
- **Ключевые слова:** 20% (точность)
- **BM25:** 10% (классическая релевантность)
