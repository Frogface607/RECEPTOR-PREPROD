# План улучшения RAG системы RECEPTOR

## Текущее состояние
- ✅ Простой текстовый поиск по файлам
- ✅ Категоризация (haccp, hr, finance, marketing, iiko)
- ✅ Базовый скоринг по совпадению слов
- ❌ Нет векторных эмбеддингов
- ❌ Нет семантического поиска
- ❌ Примитивный чанкинг

## Предлагаемые улучшения

### 1. Векторный поиск (Vector RAG)
**Приоритет: ВЫСОКИЙ**

Использовать OpenAI embeddings для семантического поиска:
- Эмбеддинги через `text-embedding-3-small` или `text-embedding-3-large`
- Хранение в векторной БД (Chroma, Qdrant, или MongoDB Atlas Vector Search)
- Семантический поиск вместо простого текстового

**Преимущества:**
- Понимает синонимы и контекст
- Находит релевантную информацию даже при разных формулировках
- Более точные результаты

### 2. Умный чанкинг
**Приоритет: СРЕДНИЙ**

- Разбиение по смыслу (по заголовкам, параграфам)
- Перекрывающиеся чанки для контекста
- Метаданные чанков (категория, источник, дата)

### 3. Гибридный поиск
**Приоритет: СРЕДНИЙ**

Комбинация:
- Векторный поиск (семантика)
- Ключевые слова (точные совпадения)
- BM25 (классический релевантность)

### 4. Реранкинг результатов
**Приоритет: НИЗКИЙ**

Использовать LLM для финального реранкинга:
- GPT-4o-mini для оценки релевантности
- Улучшение порядка результатов

### 5. Кэширование
**Приоритет: СРЕДНИЙ**

- Кэш популярных запросов
- Кэш эмбеддингов документов
- Redis для быстрого доступа

## Быстрый старт (MVP улучшений)

### Вариант 1: Простой (без векторной БД)
1. Использовать OpenAI embeddings API
2. Хранить эмбеддинги в MongoDB (как массив чисел)
3. Косинусное сходство для поиска

### Вариант 2: Продвинутый (с векторной БД)
1. Chroma DB (легковесный, встроенный)
2. Или Qdrant (производительный)
3. Или MongoDB Atlas Vector Search (если уже используем Atlas)

## Рекомендация

**Начать с Варианта 1** - быстрая реализация, значительное улучшение качества поиска.

Затем перейти на Вариант 2 для масштабирования.
