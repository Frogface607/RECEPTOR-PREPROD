# Включение живых данных — чеклист (30 минут)

> Босс, когда будет Edison apiLogin от iiko — вот точные шаги, чтобы прод
> с `receptorai.pro` показал **реальные цифры Edison** вместо демо.
> Код уже готов: всё переключается через env, без правок.

---

## 1. Где взять apiLogin (iiko Cloud)

iiko Web → личный кабинет ресторана → **Настройки → API** → создать/скопировать
**apiLogin** (он же api-key). Это одна строка. Больше ничего для Cloud не нужно —
ни логина, ни пароля.

---

## 2. Локальная проверка (5 минут)

В корне репо: `cp .env.local.example .env.local`, затем:

```
USE_MOCK_IIKO=false
EDISON_IIKO_API_LOGIN=<вставить apiLogin Edison>
```

Запусти `npm run dev` → открой `http://localhost:3000/dashboard/edison-demo`.
В подвале дашборда вместо «демо-данные» должно быть **«live iiko Cloud»**.

Если цифры пошли — переходим в прод. Если ошибка — см. §5.

---

## 3. Прод (Vercel) — env vars (5 минут)

Vercel project `receptor-preprod-final-nqu9nzl11` → Settings → Environment
Variables (Production + Preview):

```
USE_MOCK_IIKO=false
EDISON_IIKO_API_LOGIN=<apiLogin Edison>
ANTHROPIC_API_KEY=<sk-ant-...>        # включит живой AI в чате и инструментах
```

Затем **Redeploy** последний build (env подхватывается только новым деплоем).

---

## 4. Что заработает на живых данных сразу

| Раздел | Источник | Статус на реальном ключе |
|---|---|---|
| KPI: выручка, средний чек | OLAP SALES | ✅ выручка точно; чек/позиции — см. ниже |
| График выручки по дням | OLAP `OpenDate` | ✅ |
| Топ блюд | OLAP `DishName/DishGroup` | ✅ |

---

## 5. Если не подключилось

- **HTTP 401 на auth** — неверный apiLogin. Перепроверь строку, без пробелов.
- **Подвал всё ещё «демо-данные»** — `USE_MOCK_IIKO` не равно строго `false`,
  либо apiLogin пустой → код безопасно остаётся на моках (так задумано).
- **OLAP 400/500** — структура ответа отличается от ожидаемой. Это тот самый
  30-минутный момент верификации: пришли мне (в основную сессию) сырой ответ
  одного OLAP-запроса, поправлю парсинг в `src/lib/iiko/cloud-client.ts`.

---

## 6. Что нужно довести на реальном ключе (мелочи)

Эти места намеренно сделаны «приблизительно», потому что без живого ответа
iiko их нельзя проверить. На реальном ключе — быстрая доводка:

1. **Средний чек и «позиций продано» в KPI.** Сейчас в Cloud-клиенте
   `getRevenueSummary` не запрашивает число чеков. Добавить в OLAP
   `aggregateFields` поле числа чеков (`UniqOrderId.OrdersCount` или
   аналог) и пересчитать средний чек. ~15 минут.
2. **Смены по кассирам.** Cloud OLAP не отдаёт сессии кассы так, как RMS.
   Сейчас одна «смена» = один день. Если нужны настоящие смены по
   сотрудникам — потребуется RMS-канал (порт `rms-client.ts`, отдельная
   задача) или OLAP по `CashRegisterName/SessionNum`. ~1-2 часа.
3. **Поиск по меню (`найди в меню …` в чате).** Номенклатура за отдельным
   эндпоинтом со сложной нормализацией в v1. Сейчас на Cloud возвращает
   пусто (не падает). Порт — отдельная небольшая задача.

Ничего из этого не блокирует демо: выручка, топ блюд, категории, график и
AI-чат работают на живых данных сразу.

---

## 7. Откат

Если на встрече что-то пойдёт не так — поставь обратно `USE_MOCK_IIKO=true`
(или убери переменную) и Redeploy. Дашборд мгновенно вернётся на
безотказные демо-данные Edison.
