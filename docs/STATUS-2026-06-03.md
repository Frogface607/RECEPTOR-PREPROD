## Что нужно от тебя (Босс) — 3 ключа

Всё уже закодировано. Вставить в Vercel env (Production+Preview) → Redeploy:

1. **`ANTHROPIC_API_KEY=sk-ant-...`** — включает живой Claude в чате и в 27
   инструментах (сейчас они на детерминированном mock). Самый большой буст
   к «вау» на демо. Можно вставить хоть сейчас, iiko не нужен.

2. **`EDISON_IIKO_API_LOGIN=<apiLogin из iiko Web>`** + **`USE_MOCK_IIKO=false`**
   — переводит дашборд на **живые цифры Edison**. Пошаговый чеклист (10 мин
   локально + 5 мин Vercel) в `docs/REAL_CONNECT.md`. Откат — один флаг.

3. **(позже) Supabase ключи** — для Phase 3 (логин/онбординг/сохранение).
   Не блокирует демо: Михно идёт по прямой ссылке на дашборд.

## Что осознанно отложено (не блокирует)

- **Phase 3 (Supabase auth)** — по твоей инструкции не трогал без ключей.
- **Средний чек / позиции в KPI на живом iiko** — нужен один доп. OLAP-агрегат,
  проверяется только на реальном ключе (15 мин, описано в REAL_CONNECT §6).
- **Настоящие смены по кассирам** — Cloud OLAP их не отдаёт как RMS; сейчас
  «день = смена». Для настоящих нужен RMS-канал (отдельная задача).
- **Поиск по меню в чате на живом iiko** — номенклатура за отдельным
  эндпоинтом, сейчас деградирует мягко (пусто, не падает).
