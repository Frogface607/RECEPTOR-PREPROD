# 🚀 ВАРИАНТЫ ДЕПЛОЯ BACKEND

Railway закончился? Не проблема! Вот несколько вариантов:

## 🎯 РЕКОМЕНДУЕМЫЕ ВАРИАНТЫ

### 1. 🟢 Render.com (САМЫЙ ПРОСТОЙ)

**Плюсы:**
- ✅ Бесплатный tier (750 часов/месяц)
- ✅ Автоматический деплой из GitHub
- ✅ HTTPS из коробки
- ✅ Простая настройка через UI

**Минусы:**
- ⚠️ Спит после 15 минут неактивности (первый запрос медленный)

**Инструкция:** См. `DEPLOY_RENDER.md`

---

### 2. 🔵 Fly.io (БЫСТРЫЙ)

**Плюсы:**
- ✅ Бесплатный tier (3 shared-cpu VMs)
- ✅ Не спит (быстрый старт)
- ✅ Глобальная сеть (CDN)
- ✅ CLI для управления

**Минусы:**
- ⚠️ Требует установки CLI
- ⚠️ Немного сложнее настройка

**Инструкция:** См. `DEPLOY_FLY.md`

---

### 3. 💻 Локальный запуск (ДЛЯ ТЕСТИРОВАНИЯ)

**Плюсы:**
- ✅ Полный контроль
- ✅ Быстрая разработка
- ✅ Бесплатно

**Минусы:**
- ⚠️ Нужен постоянный доступ к интернету
- ⚠️ Нужно настраивать порты/файрвол

**Инструкция:** См. `LOCAL_SETUP.md`

## 🚀 БЫСТРЫЙ СТАРТ (Render.com)

1. **Создай аккаунт:** https://render.com
2. **New → Web Service** → Подключи GitHub
3. **Настройки:**
   - Root Directory: `backend`
   - Build Command: `pip install -r requirements.txt`
   - Start Command: `uvicorn server:app --host 0.0.0.0 --port $PORT`
4. **Environment Variables:**
   ```
   MONGODB_URI=...
   JWT_SECRET=...
   ENVIRONMENT=production
   ```
5. **Deploy!** 🎉

---

## 🔧 НАСТРОЙКА CORS

После деплоя, добавь URL в environment variables:

**Render.com:**
```
RENDER_SERVICE_URL=https://your-app.onrender.com
```

**Fly.io:**
```
FLY_APP_NAME=your-app-name
```

Или временно разреши все origins:
```
CORS_ALLOW_ALL=true
```
