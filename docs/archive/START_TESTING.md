# 🧪 Инструкция по запуску тестов

## ⚠️ Важно: Backend нужно запустить вручную

Из-за особенностей Windows PowerShell автоматический запуск не работает. Следуйте инструкциям ниже:

---

## 📋 Шаг 1: Запустите Backend

**Откройте первый терминал/окно PowerShell:**

```powershell
cd "C:\Users\Sergey\RECEPTOR TEST\RECEPTOR-PREPROD\backend"
python -m uvicorn server:app --host 0.0.0.0 --port 8000
```

**Дождитесь сообщения:**
```
INFO:     Uvicorn running on http://0.0.0.0:8000 (Press CTRL+C to quit)
INFO:     Started reloader process
INFO:     Started server process
INFO:     Waiting for application startup.
INFO:     Application startup complete.
```

**Оставьте это окно открытым!**

---

## 📋 Шаг 2: Запустите тесты

**Откройте второй терминал/окно PowerShell:**

```powershell
cd "C:\Users\Sergey\RECEPTOR TEST\RECEPTOR-PREPROD"
python test_batch_menu_generation.py
```

---

## ✅ Что будет протестировано:

1. **Генерация меню** - создаст тестовое меню с 6 блюдами
2. **Парсинг позиций** - распарсит меню на отдельные позиции с метаданными
3. **Конвертация в V1** - конвертирует первую позицию в V1 рецепт
4. **Конвертация в V2** - конвертирует вторую позицию в V2 техкарту
5. **Батчевая конвертация** - конвертирует несколько позиций сразу
6. **Проверка статуса** - проверит, что статусы конвертации обновились

---

## 🐛 Если что-то не работает:

### Backend не запускается:
- Проверьте, что все зависимости установлены: `pip install -r requirements.txt`
- Проверьте, что MongoDB запущен
- Проверьте, что порт 8000 свободен

### Тесты падают с ошибкой 403:
- Это нормально - нужна PRO подписка
- Тест автоматически попробует использовать `demo_user`

### Тесты падают с ошибкой подключения:
- Убедитесь, что backend запущен и отвечает на http://localhost:8000
- Проверьте в браузере: http://localhost:8000/health
