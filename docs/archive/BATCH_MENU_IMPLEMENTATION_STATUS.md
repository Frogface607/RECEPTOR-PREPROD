# 🚀 Статус реализации: Батчевая Генерация Меню V2

## ✅ ЧТО УЖЕ СДЕЛАНО

### Backend - Новые Endpoints (ГОТОВО)

1. **`GET /api/menu/{menu_id}/parse-items`** ✅
   - Парсит меню на плоский список позиций
   - Добавляет уникальные `item_id` к каждой позиции
   - Проверяет статус конвертации (V1/V2)
   - Возвращает метаданные для каждой позиции

2. **`POST /api/menu/{menu_id}/items/{item_id}/convert`** ✅
   - Конвертирует позицию меню в V1 рецепт или V2 техкарту
   - Поддерживает опции: `generate_techcard`, `save_to_history`
   - Сохраняет связь между позицией меню и созданным рецептом/техкартой

3. **`POST /api/menu/{menu_id}/items/batch-convert`** ✅
   - Батчевая конвертация нескольких позиций
   - Последовательная обработка (избегает rate limits)
   - Возвращает детальные результаты и ошибки

### Backend - Улучшения генерации меню (ГОТОВО)

1. **Автоматическое добавление `item_id`** ✅
   - При генерации меню через `/generate-menu` - добавляется `item_id` к каждому блюду
   - При генерации через `/generate-simple-menu` - также добавляется `item_id`
   - Это позволяет отслеживать каждую позицию меню

## 🐛 ИЗВЕСТНЫЕ ПРОБЛЕМЫ

1. **Проверка V1/V2 в parse-items:**
   - Сейчас используется regex для определения формата
   - Может быть неточным
   - **Решение:** Добавить поле `format` в записи user_history

2. **Батчевая конвертация:**
   - Сейчас последовательная (медленно)
   - **Решение:** Добавить параллельную обработку с ограничением

3. **Поиск позиции по item_id:**
   - Если item_id не найден, ищется по имени
   - Может быть неточным при дубликатах
   - **Решение:** Всегда использовать item_id
