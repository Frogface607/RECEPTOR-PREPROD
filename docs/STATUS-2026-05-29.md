## Архитектурные решения которые принял

1. **Mock-first строгий.** Один интерфейс `IikoClient`. `getIikoClient()`
   читает `USE_MOCK_IIKO`; default — mock. Никаких случайных пробных
   фолбэков. Чтобы перейти на реальные данные — один env var.

2. **MockIikoClient — детерминистичный.** Якорь `2026-05-29`,
   `edisonDailyRevenue(date)` детерминирована хешем даты. Те же цифры
   на каждом рендере → SSR-кэш + скриншоты для демо.

3. **AI tools — Anthropic-совместимый формат.** Каждый tool сразу
   подходит для `messages.create({ tools: [...] })`. Никакой
   подгонки потом не нужно.

4. **Cloud client порт — минимальный.** Только auth + `getRevenueSummary`.
   Остальные методы кидают чёткую ошибку с deferral. Цель — не делать
   работу, которую нельзя верифицировать без apiLogin.

5. **RMS — placeholder.** Без чейн-клиента в pipeline — порт 1094-строчного
   v1 RMS клиента это работа без покупателя. Surface есть (settings UI
   может реферить «channel: rms»), но методы кидают.

6. **Pricing UI — copy-paste из spec.** ₽9 900 Pro / ₽24 900 Team /
   ₽39 000 setup / ₽19 900 pilot. Любая корректировка — отредактировать
   `TIERS` и `FAQ` в `app/pricing/page.tsx`.

7. **Чёрный TS-only стек.** Никакого Python в репе кроме архивных
   reference-файлов в `docs/v1-iiko-reference/`. Все 100 тестов на
   Vitest, тип-safety через Zod + tsc strict.

8. **Никаких эмодзи в UI.** Уважение к brand-guide. lucide-react +
   font-display italic для всех "момента" вместо эмодзи-маркеров.

## Если что-то сломается

- **Локально `npm test` не зелёный** → читай test output, последний рабочий
  тест-снапшот на `de8c8cb9`. Не было нестабильных тестов в этой сессии.
- **Vercel build падает** → проверь Build Settings (`VERCEL_SETUP.md`).
  Если падает на Next.js команде — node version должен быть 22.x.
- **Что-то не находит модуль** → `npm install` для свежего node_modules,
  потом `npm run build`.
- **Chat вернул "Ошибка сети"** → `/api/chat` POST endpoint
  (`src/app/api/chat/route.ts`). Логи в Vercel или локально в `npm run dev`.
