# 🚀 ROADMAP: Батчевая Генерация Меню V2 - Киллер Фича

## 📋 ОБЗОР

**Цель:** Создать имбовую систему батчевой генерации меню, которая позволяет:
1. Генерировать целое меню за раз с учетом всех параметров заведения
2. Парсить меню на отдельные позиции
3. Быстро заменять любые позиции
4. Конвертировать каждую позицию в V1 или V2
5. Быстро превратить меню в техкарты и загрузить в систему

**Потенциал:** Это может стать нашей киллер-фичей, так как позволяет рестораторам:
- Создать полное меню за 2-3 минуты
- Сразу получить все техкарты
- Быстро адаптировать меню под сезон/события
- Запустить новое меню в продакшн за день

---

## 🎯 ТЕКУЩЕЕ СОСТОЯНИЕ

### ✅ Что уже работает:
1. **Генерация меню** (`/generate-menu`, `/generate-simple-menu`)
   - Учитывает профиль заведения
   - Поддерживает типы: full, seasonal, business_lunch, event
   - Создает структурированное меню с категориями и блюдами

2. **Массовая генерация техкарт** (`/generate-mass-tech-cards`)
   - Создает техкарты для всех блюд из меню
   - Последовательная генерация (без таймаутов)

3. **UI для просмотра меню**
   - Два режима: customer (для гостей) и business (анализ)
   - Замена отдельных блюд
   - Создание техкарт из блюд

4. **Конвертация V1→V2** (`/v1/convert-recipe-to-techcard`)
   - Преобразует V1 рецепт в V2 техкарту

5. **Меню V2** (`/menus.v2/generate`, `/menus.v2/{menuId}/replace`)
   - Базовая структура с заменой позиций
   - Экспорт в ZIP

### ⚠️ Что нужно улучшить:
1. **Парсинг меню на позиции** - сейчас меню структурировано, но нет удобного UI для работы с отдельными позициями
2. **Конвертация позиций в V1/V2** - нет прямого способа конвертировать позицию из меню в V1 или V2
3. **Улучшенная генерация** - нужно лучше учитывать специфику (сезонное, бизнес-ланчи)
4. **Батчевые операции** - нет возможности массово конвертировать/заменять позиции
5. **Интеграция с экспортом** - нет прямого пути от меню к экспорту в IIKO

## 🎨 АРХИТЕКТУРА РЕШЕНИЯ

### 1. Backend: Улучшенная генерация меню

#### 1.1. Новый endpoint: `/generate-menu-v2`
```python
POST /api/generate-menu-v2
{
  "user_id": "string",
  "menu_profile": {
    "menu_type": "full|seasonal|business_lunch|event",
    "dish_count": 12,
    "categories": {
      "salads": 2,
      "appetizers": 3,
      "soups": 2,
      "main_dishes": 4,
      "desserts": 2,
      "beverages": 1
    },
    "seasonal_focus": "winter|spring|summer|autumn",  # NEW
    "business_lunch_params": {  # NEW
      "price_range": "300-500",
      "preparation_time_max": 20,
      "include_soup": true,
      "include_salad": true
    },
    "special_requirements": []
  },
  "venue_profile": {...}  # Используется из профиля пользователя
}
```

**Улучшения:**
- Более умный промпт для LLM с учетом типа меню
- Специфичные инструкции для сезонного меню (использовать сезонные ингредиенты)
- Специфичные инструкции для бизнес-ланчей (быстрое приготовление, доступная цена)
- Валидация структуры меню перед сохранением

#### 1.2. Новый endpoint: `/menu/{menu_id}/parse-items`
```python
GET /api/menu/{menu_id}/parse-items?user_id={user_id}
```
Возвращает плоский список всех позиций меню с метаданными:
```json
{
  "menu_id": "string",
  "menu_name": "string",
  "items": [
    {
      "id": "string",  # Уникальный ID позиции в меню
      "index": 0,  # Индекс в меню
      "name": "string",
      "description": "string",
      "category": "string",
      "category_index": 0,
      "estimated_cost": "string",
      "estimated_price": "string",
      "main_ingredients": [],
      "difficulty": "string",
      "cook_time": "string",
      "metadata": {
        "can_convert_to_v1": true,
        "can_convert_to_v2": true,
        "has_techcard": false,
        "techcard_id": null
      }
    }
  ],
  "total_items": 12,
  "categories_count": 5
}
```

#### 1.3. Новый endpoint: `/menu/{menu_id}/items/{item_id}/convert`
```python
POST /api/menu/{menu_id}/items/{item_id}/convert
{
  "user_id": "string",
  "target_format": "v1|v2",  # В какой формат конвертировать
  "options": {
    "generate_techcard": true,  # Сразу создать техкарту
    "save_to_history": true  # Сохранить в историю
  }
}
```

**Логика:**
- Для V1: Генерирует красивый рецепт через `/v1/generate-recipe` с контекстом из меню
- Для V2: Генерирует техкарту через pipeline с контекстом из меню
- Сохраняет связь между позицией меню и созданным рецептом/техкартой

#### 1.4. Новый endpoint: `/menu/{menu_id}/items/batch-convert`
```python
POST /api/menu/{menu_id}/items/batch-convert
{
  "user_id": "string",
  "item_ids": ["id1", "id2", "id3"],  # Какие позиции конвертировать
  "target_format": "v1|v2",
  "options": {
    "generate_techcards": true,
    "save_to_history": true
  }
}
```

**Логика:**
- Параллельная конвертация (с ограничением на количество одновременных запросов)
- Возвращает прогресс и результаты

#### 1.5. Улучшенный endpoint: `/menu/{menu_id}/items/{item_id}/replace`
```python
POST /api/menu/{menu_id}/items/{item_id}/replace
{
  "user_id": "string",
  "replacement_profile": {
    "name": "string",  # Опционально - новое название
    "category": "string",  # Опционально - новая категория
    "constraints": {
      "price_range": "300-500",
      "ingredients": ["ингредиент1"],
      "avoid_ingredients": ["ингредиент2"]
    }
  }
}
```

**Логика:**
- Генерирует новое блюдо с учетом контекста меню и ограничений
- Заменяет позицию в меню
- Сохраняет историю замен

### 2. Frontend: Улучшенный UI

#### 2.1. Новый компонент: Menu Items Manager

**Расположение:** В AI-Kitchen после генерации меню

**Функционал:**
1. **Вкладка "Все позиции"**
   - Плоский список всех позиций меню
   - Фильтры: по категории, по статусу (есть техкарта/нет)
   - Поиск по названию
   - Массовый выбор позиций (чекбоксы)

2. **Вкладка "По категориям"** (текущий вид)
   - Сохраняем текущий UI

3. **Вкладка "Быстрые действия"**
   - Кнопка "Конвертировать все в V2"
   - Кнопка "Конвертировать все в V1"
   - Кнопка "Создать техкарты для всех"
   - Кнопка "Экспортировать меню в IIKO"

#### 2.2. Компонент: Menu Item Card

Для каждой позиции меню:
```
┌─────────────────────────────────────┐
│ 🍝 Паста Карбонара          [V1][V2]│
│ Категория: Основное меню            │
│ Описание: Классическая паста...     │
│ 💰 450₽ | ⏰ 20 мин | 🔥 Средне     │
│                                     │
│ [🔄 Заменить] [📋 Техкарта] [⚙️]   │
│                                     │
│ Статус: ⚪ Нет техкарты             │
│         ✅ V1 рецепт создан         │
│         ✅ V2 техкарта создана      │
└─────────────────────────────────────┘
```

**Действия:**
- **🔄 Заменить** - открывает модалку замены
- **📋 Техкарта** - создает техкарту (V2)
- **📝 V1** - конвертирует в V1 рецепт
- **⚙️** - меню дополнительных действий

#### 2.3. Модалка: Batch Operations

При выборе нескольких позиций:
```
┌─────────────────────────────────────┐
│ Массовые операции (3 позиции)       │
│                                     │
│ ○ Конвертировать в V1 рецепты       │
```
